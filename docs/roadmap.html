Freedom — Strategic Roadmap

Phase plan for a modern browser that stays zero-trust, zero-knowledge, and pure C.

Legend

Each milestone follows the strict cycle: Spec → Test (red) → Code (green) → Refactor → Validation (ASan) → Fuzzing → Docs. A milestone is closed only when make test && make asan passes.

  • ✓ Closed — spec + tests + ASan + fuzzing
  • In progress — active development
  • Pending — not started, awaiting dependencies

Current State (closed July 2026)

  • 52 modules — 70,187 lines of C (src/) + 11,540 (gui/)
  • 1,317 tests across 49 test files — CMocka + ASan/UBSan
  • 16 fuzz targets — libFuzzer + AFL++
  • Full pipeline: Network (TLS 1.3 + PQ X25519MLKEM768) → Parser (Lexbor) → JS (QuickJS-ng) → CSS (4K-line pure parser) → Layout (flex/grid/float) → Compositor (z-index, opacity, mix-blend, transform) → Paint (Cairo + HarfBuzz)
  • Sandbox: fork+exec worker, seccomp W^X, Landlock, CLONE_NEWNET, anti-fingerprint identity, canvas/audio poisoning per-origin
  • Media: PNG/JPEG/WebP/GIF/AV1 decode → worker, FFmpeg video/audio pipeline, HLS, data: URIs
