Box model, flex and grid

This page exercises the structured layout engine: per-tag user-agent margins (box_style), a flex row, and a grid. Layout is structure, so it applies by default — you do NOT need to enable author colors (CSS) for the columns to show. Only author colors are gated behind the options menu.

A flex row (three columns)

First column. Each direct block becomes one flex item, sized by the solver and placed side by side instead of stacking vertically.

Second column. The gap between items and the justify-content distribution come straight from the inline style on the container.

Third column. Wrapping happens inside each column at its resolved width.

A grid (two equal columns)

Grid cell one. Two equal tracks (1fr each), filled row by row.

Grid cell two. The second track on the first row.

Grid cell three. Wraps to the next row, first track.

Grid cell four. Second track, second row.

Heading hierarchy and paragraph spacing

An h3 sits closer than an h2

Paragraphs get their user-agent top/bottom margin (1em), and adjacent margins collapse, so the vertical rhythm matches a real browser's defaults.

An h4 is tighter still
